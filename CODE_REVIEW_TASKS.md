# Предложенные задачи по итогам быстрого ревью

## 1) Исправление опечатки/терминологии
**Задача:** унифицировать термин «эндпоинт» в комментариях `backend/app.py` (сейчас используется форма «Эндпоинты»); выбрать один стиль написания и применить его во всех секциях файла.

**Почему это важно:** снижает визуальный шум и делает кодовую базу более единообразной для команды.

**Где видно:** заголовки секций в `backend/app.py`.

---

## 2) Исправление ошибки (bugfix)
**Задача:** добавить валидацию `request.get_json()` на `None` во всех POST/PUT-обработчиках (`/api/yarns`, `/api/samples`, `/api/projects`), чтобы вместо `500` возвращать `400` с понятным сообщением при пустом/невалидном JSON.

**Почему это ошибка:** сейчас код сразу итерируется по `data` или обращается к `data.get(...)`; если тело запроса пустое, сервер выбрасывает исключение и уходит в общий `except` с `500`.

---

## 3) Исправление комментария / несоответствия документации
**Задача:** исправить вводящий в заблуждение комментарий в `backend/models.py`: сейчас указано, что база заполняется «если она пуста», но проверка на пустоту не реализована.

**Почему это важно:** комментарий обещает поведение, которого нет; это может привести к дублированию данных при повторном запуске скрипта.

---

## 4) Улучшение теста
**Задача:** добавить интеграционный тест на Flask API: `POST /api/yarns` с пустым телом и `Content-Type: application/json` должен возвращать `400` и диагностическое сообщение (после bugfix из п.2).

**Почему это улучшение:** тест закрепит ожидаемое поведение по валидации входных данных и защитит от регрессии в обработке ошибок.
